Overview

A compact handheld interface for RC vehicles

This PCB is a compact remote control interface designed for RC cars, rovers, and other small robotic vehicles. It was developed under IEEE at UC Irvine and was designed using KiCad to provide a reliable, handheld interface.

The board uses a WeAct ESP32-C3 microcontroller from WeAct Studio, powered via USB, and features two LEDs (moving and stopped indicators) plus an XY joystick with an integrated push-button switch. This setup provides a simple, responsive interface for hobbyist robotics and RC projects.

📎 3D render and assembled board photos coming soon.

01 — Features

Board Capabilities

  • USB-powered WeAct ESP32-C3 from WeAct Studio (5V)
  • Dual LED indicators for motion state (with built-in current-limiting resistors)
  • XY analog joystick with integrated switch
  • Compact, handheld-friendly PCB
  • Compatible with RC cars, rovers, and small robots

02 — Components

Hardware Breakdown

Microcontroller

WeAct ESP32-C3 from WeAct Studio (powered via USB). Note: The ESP32-C3 library is required in KiCad for this design.

Input Device

XY analog joystick with pins labeled VRX, VRY, and SW (switch).

Indicators

2x indicator LEDs with built-in current-limiting resistors:

  • LED1: Moving indicator (lights when vehicle is moving)
  • LED2: Stopped indicator (lights when vehicle is stationary)

Power Supply

5V from USB connection.

04 — Usage

Operating Instructions

  • Connect the WeAct ESP32-C3 from WeAct Studio to a USB power source (5V)
  • The ESP32-C3 initializes and sets the LED states according to movement commands
  • Move the joystick to control the vehicle:
    • Forward/Backward → Vehicle motion
    • Left/Right → Steering
  • LED indicators reflect movement state automatically:
    • Moving LED ON when the vehicle moves
    • Stopped LED ON when stationary
  • Press the joystick switch for additional programmed functions

05 — Programming

Software Implementation

The firmware should implement the following logic:

  • Read the analog joystick values from VRX and VRY
  • Read the digital state from SW
  • Map joystick input to RC commands (PWM, motor driver, or wireless protocol)
  • Update LEDs based on the motion state

06 — Applications

Use Cases

  • RC cars, trucks, and rovers
  • Remote-controlled robotics projects
  • Educational STEM projects
