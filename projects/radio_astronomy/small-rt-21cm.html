Overview

Detecting galactic hydrogen from the backyard

This project is a ground-up radio telescope designed to detect the 21 cm hydrogen line — a spectral emission at 1420.405 MHz produced by neutral hydrogen atoms throughout the galaxy. Observing this line is one of the most fundamental techniques in radio astronomy, and the distribution of hydrogen across the Milky Way carries direct evidence for the large-scale structure of the universe.

The telescope uses a 2.2 m parabolic dish on a custom altitude-azimuth mount, pointed by a motorized drive system. The project is divided into two distinct subsystems: the RF signal chain that captures and conditions the hydrogen line signal, and the mechanical drive system that controls pointing.

01 — RF Electronics

Signal Chain

The signal chain is a multi-stage analog receive path that amplifies, filters, and frequency-converts the faint hydrogen line emission down to a baseband signal the RTL-SDR can digitize. Each stage is chosen to maximize sensitivity and selectivity at 1420 MHz while rejecting out-of-band interference.

IQ Mixer & the Local Oscillator

The image rejection mixer is the heart of the receiver's frequency conversion. A local oscillator (LO) running at 1416 MHz is mixed with the incoming 1420 MHz signal, which mathematically produces sum and difference frequencies — the difference lands the hydrogen line at roughly 4 MHz, a frequency the RTL-SDR can easily digitize. An "image rejection" mixer generates two copies of the mixed signal 90° apart in phase (the I and Q channels), then combines them so that signals at the image frequency — which would otherwise fold in on top of the desired signal — cancel out. This gives a clean, interference-free baseband that represents only the sky around 1420 MHz.

Testing & Validation

Prior to integrating the full signal chain, individual stages were validated on the bench using a spectrum analyzer and signal generator at 1420 MHz. This allowed verification of filter passband, LNA gain, and connector integrity before committing to a full system test on the dish.

02 — Mechanical

Altitude-Azimuth Mount

The dish is mounted on a custom altitude-azimuth (alt-az) structure that provides independent control of elevation (altitude) and horizontal rotation (azimuth), enabling the telescope to be pointed at any position in the sky. The drive system is motorized, allowing the mount to compensate for Earth's rotation and track targets over time.

  • 2.2 m parabolic dish providing sufficient gain to resolve faint galactic hydrogen emission.
  • Altitude axis driven by a linear actuator controlled via joystick input — elevation is set and held by the actuator's physical stroke.
  • Azimuth axis allows full 360° rotation for sky coverage.
  • Drive electronics use dual VNH5019 motor driver ICs with current sensing for stall detection and fault protection.

03 — Motor Control

Altitude Drive Firmware

The altitude axis is driven by a linear actuator controlled from an Arduino via a joystick. The firmware maps joystick deflection to motor PWM while implementing a deadzone at center, driver fault detection via the VNH5019's EN/DIAG pin, and a stall-current latch that cuts power if sustained overcurrent is detected and requires the operator to return the joystick to center before re-engaging.

altitude-motors.ino C++ 
// VNH5019 pulls EN/DIAG LOW on a driver fault — check before anything else
if (digitalRead(M1_EN) == LOW || digitalRead(M2_EN) == LOW) {
    stopMotors();
    Serial.println("DRIVER FAULT!");
    return;
}

// check both motors for sustained overcurrent
if (!stalled &&
    (checkStall(M1_CS, stallStartM1) ||
     checkStall(M2_CS, stallStartM2))) {

    Serial.println("STALL CURRENT!");
    stopMotors();
    stalled = true; // latch — operator must center joystick to re-engage
}

// map joystick deflection to PWM with deadzone at center
if (abs(delta) < DEADZONE)      { stopMotors(); }
else if (delta > 0)              { pwmValue = map(delta,  DEADZONE, 512, 0, MAXSPEED); forwardMotors(pwmValue); }
else                             { pwmValue = map(-delta, DEADZONE, 512, 0, MAXSPEED); reverseMotors(pwmValue); }

Safety Features

  • Driver fault detection reads the VNH5019 EN/DIAG open-drain pin — held HIGH by INPUT_PULLUP and pulled LOW by the IC on any over-temperature or under-voltage event.
  • Stall detection monitors the current sense (CS) pin on both motor channels; sustained high current triggers a latched stop to protect the actuator and driver.
  • Stall latch requires the joystick to return to center before the drive can be re-engaged, preventing an accidental re-start into an already-stalled load.
  • Configurable DEADZONE constant prevents the actuator from creeping due to joystick center offset.
  • MAXSPEED constant caps PWM output to limit mechanical stress on the linear actuator.
